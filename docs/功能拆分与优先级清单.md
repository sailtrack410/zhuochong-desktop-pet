# 桌宠 AI 功能拆分与优先级清单

## 1. 文档目的

本文档用于将桌宠 AI 项目的需求拆分为可执行的功能模块，并明确各模块的优先级、依赖关系和推荐开发顺序。

本文档不强制绑定开发周期，适用于自由开发节奏。

本文档目标：

- 将需求拆分为清晰功能块
- 标记必须项、增强项、可延后项
- 明确模块依赖关系
- 降低后续开发过程中的方向漂移

## 2. 优先级定义

为避免后续“想到什么做什么”，本文档将功能分为四级：

### P0：基础成立功能

没有这些功能，项目核心体验不成立。

### P1：首版完整体验功能

没有这些功能，桌宠能运行，但体验不完整。

### P2：增强体验功能

有助于提升角色感、实用性和稳定性，但不是第一阶段阻塞项。

### P3：长期扩展功能

适合后续版本逐步补充，不建议前期投入过多。

## 3. 总体开发原则

- 先做“存在感”，再做“智能感”
- 先做“可持续常驻”，再做“功能丰富”
- 先做“稳定的少功能”，再做“复杂的多功能”
- 先做跨平台抽象，再补平台特化能力
- 先做默认角色和基础 UI，再优化视觉细节

## 4. 功能地图

项目可拆分为以下一级模块：

1. 桌宠外壳模块
2. 基础交互模块
3. 聊天模块
4. 提醒与主动行为模块
5. 记忆与日记模块
6. 系统感知模块
7. 设置与配置模块
8. 托盘与应用控制模块
9. 数据与本地存储模块
10. 跨平台适配模块
11. UI 风格与资源模块
12. 调试与可观测性模块

## 5. 功能拆分与优先级

## 5.1 桌宠外壳模块

### 目标

让桌宠稳定存在于桌面上，并具备基础角色状态显示能力。

### 功能项

#### P0

- 创建桌宠窗口
- 默认只显示小猫本体
- 支持窗口置顶
- 支持透明背景显示
- 支持桌宠位置记忆
- 支持基础状态资源切换

#### P1

- 支持靠边停靠
- 支持窗口缩放或像素倍数缩放
- 支持显示/隐藏切换

#### P2

- 支持更自然的停靠反馈
- 支持桌面多显示器位置记忆

#### P3

- 支持多角色切换

### 依赖

- UI 资源模块
- 数据与本地存储模块

## 5.2 基础交互模块

### 目标

让用户可以自然地操作桌宠，并得到反馈。

### 功能项

#### P0

- 鼠标点击小猫
- 点击后弹出小聊天框
- 支持拖拽移动
- 支持点击反馈状态变化

#### P1

- 点击空白处关闭聊天框
- 支持基础 hover 反馈
- 支持拖拽中的状态变化

#### P2

- 支持双击快捷动作
- 支持右键菜单快捷入口

#### P3

- 支持更复杂的肢体互动或小游戏式反馈

### 依赖

- 桌宠外壳模块
- 聊天模块

## 5.3 聊天模块

### 目标

让桌宠具备基础陪伴式文字交流能力。

### 功能项

#### P0

- 小聊天框 UI
- 用户输入文本
- 发送消息
- 显示桌宠回复
- 接 OpenAI 兼容接口
- 用户自行配置模型地址、模型名、API Key
- 基础会话上下文

#### P1

- 聊天历史显示
- 失败重试提示
- 基础加载状态
- 角色化回复风格约束

#### P2

- 多轮上下文优化
- 会话裁剪策略
- 回复类型区分，例如闲聊、提醒、安慰

#### P3

- 多人格切换
- 多模型切换

### 依赖

- 设置与配置模块
- 数据与本地存储模块

## 5.4 提醒与主动行为模块

### 目标

让桌宠在适当时机主动说话或提醒，但不过度打扰。

### 功能项

#### P0

- 主动性强度配置
- 默认中等主动
- 定时检查是否需要提醒
- 基础提醒弹出
- 静默规则：全屏闭嘴
- 静默规则：专注/免打扰时闭嘴

#### P1

- 冷却时间控制
- 提醒类型区分
- 轻提醒与强提醒分级
- 支持基于系统感知的提醒触发

#### P2

- 更细的行为策略
- 更自然的陪伴型主动发言
- 不同情境下不同口吻

#### P3

- 基于长期关系变化调整主动性

### 依赖

- 系统感知模块
- 设置与配置模块
- 记忆与日记模块

## 5.5 记忆与日记模块

### 目标

让桌宠逐渐认识用户，并把重要内容沉淀为长期可查看信息。

### 功能项

#### P0

- 从聊天中自动提取记忆候选
- 生成结构化记忆数据
- 每天沉淀为日记
- 提供记忆/日记查看页面
- 软件内只读
- 本地文件可编辑

#### P1

- 记忆分类
- 记忆去重
- 记忆可信度标记
- 记忆与日记分开展示

#### P2

- 记忆摘要
- 周期性归档
- 记忆与聊天联动引用

#### P3

- 关系成长系统
- 记忆图谱

### 依赖

- 聊天模块
- 数据与本地存储模块

## 5.6 系统感知模块

### 目标

为提醒和聊天提供环境上下文。

### 功能项

#### P0

- 获取时间
- 获取电量
- 获取天气
- 获取系统空闲/活跃状态
- 标准化输出感知上下文

#### P1

- macOS 读取系统日历
- Windows 平台保留空实现或替代实现
- 感知结果缓存

#### P2

- 更细粒度的环境分析
- 异常状态处理

#### P3

- 前台应用识别映射优化
- 更多系统服务接入

### 依赖

- 跨平台适配模块
- 数据与本地存储模块

## 5.7 设置与配置模块

### 目标

让用户可以完成接入和行为调整。

### 功能项

#### P0

- API Key 配置
- Base URL 配置
- 模型名称配置
- 主动性档位配置
- 基础行为设置

#### P1

- 平台权限提示
- 天气配置
- 默认提醒偏好

#### P2

- 导入导出配置
- 多套配置切换

#### P3

- 在线配置同步

### 依赖

- 数据与本地存储模块

## 5.8 托盘与应用控制模块

### 目标

让用户以桌面应用方式控制桌宠。

### 功能项

#### P0

- 托盘图标
- 打开设置
- 显示/隐藏桌宠
- 退出应用

#### P1

- 打开记忆/日记
- 快速切换静音或免打扰

#### P2

- 托盘快速查看当前状态

#### P3

- 托盘快捷预设切换

### 依赖

- 设置与配置模块
- 桌宠外壳模块

## 5.9 数据与本地存储模块

### 目标

为配置、聊天、记忆和状态提供稳定的本地数据基础。

### 功能项

#### P0

- 配置文件存储
- 聊天记录存储
- 记忆数据存储
- 日记数据存储
- 桌宠位置存储

#### P1

- 数据版本管理
- 基础迁移能力
- 本地文件结构规范

#### P2

- 数据清理策略
- 历史归档策略

#### P3

- 更复杂的数据索引

- 无，是底层模块

## 5.10 跨平台适配模块

统一抽象 Windows 与 macOS 的平台能力差异。

### 功能项

#### P0

- 平台识别
- 通用接口抽象
- 时间能力统一
- 电量能力统一
- 空闲/活跃状态能力统一

#### P1

- macOS 日历适配
- Windows 空实现或替代实现说明
- 平台能力可用性判断

#### P2

- 平台差异日志
- 平台降级策略

#### P3

- 更复杂的平台能力统一层

### 依赖

- 系统感知模块

#### P1

- 应用图标

- 基础 UI 视觉规则

#### P2

- 更完整的状态资源

### 依赖

- 桌宠外壳模块
- 基础交互模块

### 目标

### 功能项

- 基础日志

- 配置加载日志

#### P1

- 调试面板

- 最近提醒记录

#### P2

- 平台能力检测页

## 6. 推荐开发顺序

在自由开发模式下，建议按依赖和收益排序：

### 第一阶段：让桌宠存在

- 桌宠外壳模块 P0
- 基础交互模块 P0
- UI 风格与资源模块 P0
- 数据与本地存储模块 P0 中最基础部分

目标：

- 小猫能出现在桌面上
- 能点击
- 能拖拽
- 能记住位置

### 第二阶段：让桌宠能聊

- 聊天模块 P0
- 设置与配置模块 P0
- 调试与可观测性模块 P0

目标：

- 能配置 API
- 能弹出小聊天框
- 能稳定对话

### 第三阶段：让桌宠会主动

- 系统感知模块 P0
- 跨平台适配模块 P0
- 提醒与主动行为模块 P0

目标：

- 能感知环境
- 能决定何时提醒
- 能在不打扰的前提下主动说话

### 第四阶段：让桌宠会记住你

- 记忆与日记模块 P0
- 数据与本地存储模块后续能力

目标：

- 自动提取记忆
- 每日形成日记
- 软件内可查看

### 第五阶段：补全完整体验

- 托盘与应用控制模块 P1
- 聊天模块 P1
- 提醒与主动行为模块 P1
- 系统感知模块 P1
- UI 风格与资源模块 P1

目标：

- 体验更完整
- 使用更顺手

## 7. 模块依赖关系

可以简化理解为：

- 桌宠外壳模块 是界面基础
- 基础交互模块 依赖外壳模块
- 聊天模块 依赖设置模块和存储模块
- 提醒模块 依赖系统感知、设置、记忆
- 记忆模块 依赖聊天和存储
- 系统感知模块 依赖跨平台适配
- 托盘模块 依赖外壳和设置
- 调试模块横跨所有模块

## 12. 当前应避免提前投入的内容

以下内容目前不建议提前投入时间：

- 复杂语音系统
- 自动操作电脑
- 高复杂度动画系统
- 过早追求视觉完美
- 复杂插件体系
- 多代理架构

## 14. 结论

本项目后续开发应按“先成立、再完整、再增强、最后扩展”的顺序推进。

最重要的是先把以下闭环做出来：

- 小猫常驻桌面
- 点击可聊
- 能适度提醒
- 能慢慢记住你

后续新增功能都应对照本清单判断其优先级，避免打乱主线。
