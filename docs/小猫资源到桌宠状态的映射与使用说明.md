# 小猫资源到桌宠状态的映射与使用说明

## 1. 文档目的

本文档用于说明项目当前引入的 `Cat_player` 小猫资源应该如何用于桌宠 MVP，包括：

- 原始资源结构说明
- 推荐状态映射
- 哪些动作适合桌宠
- 哪些动作先不要接入
- 前端资源目录的标准命名建议

本文档基于当前项目目录中的资源：

- `Cat_player/Cat_sheets`
- `Cat_player/gif`

## 2. 当前资源类型说明

这套资源不是 Live2D，也不是单张立绘，而是一套像素风小猫动作资源。

包含两类文件：

### 2.1 `Cat_sheets`

这一层是横向精灵图。

特点：

- 每张 PNG 高度统一为 `32px`
- 宽度为 `32 * 帧数`
- 例如 `96x32` 表示 3 帧
- 例如 `160x32` 表示 5 帧

适合场景：

- 需要自己按帧播放
- 需要后续改成 Canvas 动画
- 需要更精细控制播放速度

### 2.2 `gif`

这一层是已经导出的 GIF 动画。

特点：

- 直接可显示
- 单帧尺寸统一为 `32x32`
- 适合先接入桌宠 MVP

结论：

首版优先使用 `gif` 目录，不要先处理 `Cat_sheets`。

## 3. 资源风格判断

这套资源的风格是：

- 像素风
- 游戏角色动作包
- 小体积低成本
- 很适合作为桌宠原型资源

不属于：

- 插画风桌宠
- Live2D 角色
- 高分辨率品牌角色

因此它非常适合当前阶段：

- 快速验证桌宠功能
- 先完成窗口、交互、状态切换、气泡和对话
- 暂时不追求最终正式美术

## 4. 推荐桌宠状态映射

桌宠首版建议只保留少量高频状态，不要把游戏动作全都接进去。

推荐映射如下：

| 桌宠状态 | 建议资源 | 用途 |
| --- | --- | --- |
| `idle` | `gif/CAT_idle_1.gif` | 默认待机 |
| `sleep` | `gif/CAT_asleep_1.gif` | 安静模式、夜间、空闲过久 |
| `move` | `gif/CAT_walk_1.gif` | 桌面短距离移动 |
| `run` | `gif/CAT_run_1.gif` | 快速移动、追逐光标等 |
| `drag` | `gif/CAT_ducking_idle_1.gif` | 被用户拖拽时 |
| `click` | `gif/CAT_win_cheer_1.gif` | 点击反馈、收到夸奖 |
| `hurt` | `gif/CAT_hit_1.gif` | 被打扰、拒绝时的短反馈 |
| `hide` | `gif/CAT_against_wall.png` | 靠边停靠、缩在角落 |

## 5. 聊天状态建议

这套资源里没有明显的“说话”动作，所以不要强行给它做口型动画。

首版建议：

- 聊天时仍然保持 `idle`
- 通过气泡文字表达“说话”
- 可在聊天回复出现时短暂切换到 `click`

也就是说：

- `talk -> idle`
- `reply-highlight -> click`

这样足够了。

## 6. 当前不建议接入的动作

以下动作更像游戏行为，不适合作为首版桌宠常规状态：

- `CAT_attack_1.gif`
- `Cat_dead_1.gif`
- `CAT_jump_1.gif`
- `CAT_ladder_1.gif`
- `CAT_spining_1.gif`
- `CAT_ducking_move_1.gif`
- `CAT_LANDING.gif`
- `CAT_fall_2.png`

原因：

- 和桌面陪伴场景不匹配
- 会让角色行为显得过激或奇怪
- 增加状态切换复杂度

这些资源可以后续用于彩蛋或特殊交互，不要在 MVP 主流程里使用。

## 7. 桌宠首版最小状态集合

如果只做 MVP，建议先只用这 5 个：

- `idle`
- `sleep`
- `drag`
- `click`
- `move`

这已经足够支持：

- 默认常驻
- 被点击
- 被拖动
- 空闲睡觉
- 在桌面上偶尔移动

## 8. 推荐前端资源目录结构

建议不要直接在代码里引用 `Cat_player/gif/CAT_idle_1.gif` 这种原始名字。

建议整理成标准目录：

```text
assets/
  pets/
    pixel-cat/
      raw/
      runtime/
        states/
          idle.gif
          sleep.gif
          move.gif
          run.gif
          drag.gif
          click.gif
          hurt.gif
          hide.png
      manifest.json
      README.md
```

说明：

- `raw/`：放原始素材，不修改命名
- `runtime/states/`：放项目真正使用的标准命名文件
- `manifest.json`：前端读取状态配置
- `README.md`：记录来源和映射关系

## 9. 推荐状态命名规则

为了后续方便扩展，不要把状态命名成和素材文件名一样。

建议统一用业务状态名：

- `idle`
- `sleep`
- `move`
- `run`
- `drag`
- `click`
- `hurt`
- `hide`

如果以后扩展聊天和情绪，也继续沿用这种命名：

- `happy`
- `confused`
- `angry`
- `thinking`

不要把资源命名混进业务逻辑。

## 10. Manifest 建议结构

建议做一个简单配置文件：

```json
{
  "characterId": "pixel-cat",
  "displayName": "Pixel Cat",
  "frameSize": {
    "width": 32,
    "height": 32
  },
  "defaultState": "idle",
  "states": {
    "idle": {
      "src": "./runtime/states/idle.gif",
      "loop": true
    },
    "sleep": {
      "src": "./runtime/states/sleep.gif",
      "loop": true
    },
    "move": {
      "src": "./runtime/states/move.gif",
      "loop": true
    }
  }
}
```

这样后面前端只要根据状态读取配置即可。

## 11. 前端使用建议

首版接入方式建议非常简单：

1. 应用启动时加载 `manifest.json`
2. 根据当前桌宠状态找到资源路径
3. 用 `<img>` 或图片组件直接渲染
4. 状态变化时切换图片源

首版不要做复杂帧控制，直接播放 GIF 即可。

## 12. 推荐状态切换规则

建议首版采用以下简单规则：

- 默认：`idle`
- 长时间无交互：`sleep`
- 用户点击：`click`，1 秒后回到 `idle`
- 用户拖拽：`drag`
- 程序移动角色位置：`move`
- 角色快速位移：`run`
- 靠边停靠：`hide`

## 13. 本资源的定位

这套资源更适合充当：

- 占位版本角色
- MVP 测试资源
- 交互验证素材

不一定适合作为后续正式商业版主角色，但非常适合作为开发期可用资源。

## 14. 执行建议

当前最合理的下一步是：

1. 保留 `Cat_player` 作为原始素材目录
2. 复制一份精选状态到 `assets/pets/pixel-cat/runtime/states`
3. 用标准命名重命名
4. 新建 `manifest.json`
5. 后续桌宠前端只读取标准目录，不直接读原始素材目录

这也是本文档后续配套执行的目录整理方案。
